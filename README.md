# Библиотечная система управления

**Веб-сайт для управления библиотечным фондом, учёта книговыдачи и обслуживания читателей.** Создано на Go с современным веб-интерфейсом. Позволяет библиотекарям и читателям организовывать библиотечные процессы, отслеживать книговыдачу, управлять задолженностями и мотивировать пользователей системой читательских билетов.

---

## О проекте

### Почему был создан этот продукт?

Library Management System появилась как практическая реализация цифровой системы управления библиотекой в формате веб-приложения. Проект объединяет потребности читателей в поиске и бронировании книг, отслеживании своей истории чтения с инструментами библиотекаря для управления фондом, выдачей и контролем возвратов. Разработка велась как комплексный пример полноценного корпоративного приложения с архитектурой REST API, реляционной базой данных и современным интерфейсом.

### Какую проблему он решает?

Приложение предоставляет единую цифровую среду для организации библиотечных процессов без зависимости от облачных сервисов. Оно автоматизирует рутинные операции: ведение каталога, выдачу книг, контроль сроков возврата, управление читательскими билетами и отслеживание задолженностей, заменяя разрозненные картотеки и бумажные формуляры структурированной системой.

### В чём его ключевые преимущества?

*   **Полная локальность и безопасность**: Все данные хранятся в локальной базе PostgreSQL на вашем компьютере
*   **Ролевая система**: Разграничение прав доступа для читателей и администраторов
*   **Комплексное управление библиотекой**: Книги, пользователи, выдача, возврат, уведомления в одном приложении
*   **Мотивация через читательский билет**: Электронный билет с трекером прочитанных книг и статистикой
*   **Социальные функции**: Системные уведомления для всех пользователей
*   **Готовое рабочее решение**: Полнофункциональное приложение с типовыми библиотечными сценариями «из коробки»

## Технологии

Для работы приложения требуется:

*   **Go** (Golang) 1.21 или выше — бэкенд и API
*   **PostgreSQL** 14.0 или выше — база данных
*   **Node.js** 16.0 или выше — фронтенд (React)

Архитектура проекта:

| Компонент | Технология |
|---|---|
| Язык бэкенда | Go 1.21 с фреймворком Gin |
| База данных | PostgreSQL 14.0 с полной схемой пользователей, книг, выдач и уведомлений |
| Архитектура | MVC с чётким разделением: маршруты, обработчики, модели, база данных |
| Фронтенд | React 18 с современным UI (Material-UI и custom CSS) |
| Безопасность | JWT-аутентификация, шифрование паролей (bcrypt), защита от SQL-инъекций |
| Дополнительные компоненты | Генерация CSV-отчетов, загрузка аватаров, система уведомлений |

## Быстрая установка и запуск

Вы можете запустить приложение за несколько минут, используя готовые файлы:

### 1. Установите необходимое ПО

*   Установите **Go** по ссылке: [golang.org/doc/install](https://golang.org/doc/install)
*   Установите **PostgreSQL** по ссылке: [postgresql.org/download](https://www.postgresql.org/download/)
*   Установите **Node.js** по ссылке: [nodejs.org](https://nodejs.org/)

### 2. Настройте базу данных

```bash
# Создайте базу данных
createdb library_db

# Импортируйте схему (файлы из internal/database/db.go уже содержат создание таблиц)
# При первом запуске приложения таблицы создадутся автоматически
```

### 3. Настройте переменные окружения

Создайте файл `.env` в корневой папке:

```env
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=ваш_пароль
DB_NAME=library_db
JWT_SECRET=ваш_секретный_ключ
PORT=8080
```

### 4. Запустите приложение

**Запуск бэкенда:**
```bash
cd library-management
go mod tidy
go run main.go
```

**Запуск фронтенда (в отдельном терминале):**
```bash
cd frontend
npm install
npm start
```

> **Важно**: Перед запуском убедитесь, что сервис PostgreSQL запущен в вашей системе.

## Возможности

Library Management System предоставляет полный цикл управления библиотекой:

*   **Система пользователей**: Регистрация и авторизация с ролями `user` и `admin`, изоляция данных
*   **Управление книгами**: Создание карточек книг с названием, автором, ISBN, издательством, годом и количеством копий
*   **Выдача и возврат**: Оформление выдачи книг читателям с контролем доступных экземпляров и автоматическим обновлением количества
*   **Продление книг**: Возможность продления срока возврата на 14 дней
*   **Статистика и аналитика**: Подсчёт активных выдач, просроченных книг и общего количества прочитанного
*   **Электронный читательский билет**: Привязка пластикового билета к профилю с отображением статуса и статистики
*   **Администрирование**: Управление пользователями (блокировка, смена роли), просмотр должников
*   **Уведомления**: Системные уведомления для всех пользователей и личные оповещения

## Использование

### Начало работы

При первом запуске зарегистрируйтесь как новый пользователь, или войдите под существующими данными. Первый зарегистрированный пользователь автоматически получает роль `admin`.

### Главный экран

После входа откроется рабочее пространство с доступом к статистике, книгам, билету и уведомлениям.

### Основные функции (панель навигации)

| Раздел | Описание |
|---|---|
| **Главная** | Просмотр личной статистики (активные выдачи, просрочки, прочитано книг) |
| **Профиль** | Редактирование личных данных, привязка билета, загрузка аватара |
| **Мои книги** | Просмотр истории выдач, продление активных книг |
| **Уведомления** | Просмотр личных уведомлений системы |
| **Календарь** | Визуальное отображение дат выдачи и возврата |
| **Билет** | Просмотр электронного читательского билета |
| **Админ панель** | Управление пользователями, должниками, отчётами и системными уведомлениями |

### Работа с данными

*   Двойной клик по карточке книги или пользователя открывает детальную информацию
*   Все изменения сохраняются автоматически через транзакции БД
*   Фильтрация расписания по текущему дню для быстрого доступа

## Разработка и тестирование

Проект построен на архитектуре MVC с чётким разделением:

*   **Модели**: `User`, `Book`, `BookLoan`, `Notification`, `SystemNotification` и др.
*   **Контроллеры (Handlers)**: `auth.go`, `book.go`, `admin.go`, `notification.go`, `report.go`, `stats.go`, `ticket.go`, `user.go`
*   **Маршруты**: Централизованное определение в `main.go`
*   **DAO-слой**: Прямая работа с БД через `database/sql` в обработчиках

### Тестирование

Проведено комплексное тестирование, охватывающее основные сценарии использования:

| Тест-кейс | Сценарий | Охватываемый функционал |
|---|---|---|
| T1 | Полная регистрация и вход | Аутентификация, создание профиля |
| T2 | Минимальный путь использования | Базовая авторизация, загрузка интерфейса |
| T3 | Работа с профилем | Редактирование, привязка билета |
| T4 | Управление книгами | История выдач, продление книг |
| T5 | Статистика и календарь | Просмотр прогресса и событий |
| T6 | Электронный билет | Отображение статуса и статистики |
| T7 | Администрирование | Управление пользователями, блокировка |
| T8 | Отчёты и уведомления | Генерация CSV, системные оповещения |
| T9–T13 | Навигация по всем модулям | Переходы между вкладками, фильтры |

Интеграционное тестирование проводилось на основе графа переходов между окнами приложения, обеспечивая покрытие всех ключевых пользовательских сценариев.
